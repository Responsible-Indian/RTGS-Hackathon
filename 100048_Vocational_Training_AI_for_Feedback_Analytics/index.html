Root Cause Analysis

The Anatomy of Signal Loss

Understanding WHY student feedback fails to become institutional intelligence. These systemic bottlenecks explain why problems persist despite data being collected.

01

The Language Processing Gap

Telugu, English, trade jargon, code-mixingno system can parse it.

ITI students express feedback in Telugu, English, or a fluid mix of bothoften incorporating colloquial expressions and trade-specific terminology. Without specialized NLP capabilities for regional languages, automated processing cannot extract meaning from this linguistic diversity.

Systemic Root Causes
  • No bilingual NLP models trained for Telugu-English code-mixing
  • No trade-term lexicons for vocational contexts
  • Generic tools fail on sarcasm, idioms, colloquialisms
Open-text feedback remains unanalyzable, insights locked in raw text
02

The Processing Pipeline Void

Data flows in, but no intelligence flows out.

Between raw feedback collection and actionable insight lies nothing. No sentiment analysis pipeline detects student satisfaction. No comprehension scoring identifies struggling learners. No LLM-based summarization distills patterns from individual responses.

Systemic Root Causes
  • No sentiment classification infrastructure exists
  • No anomaly detection for early warning signals
  • No automated summarization of thousands of entries
Sentiment trends, comprehension levels, dissatisfaction patternsall invisible
03

The Integration Chasm

Systems that don't talk cannot coordinate action.

Student feedback apps, ITI Training Management Systems, and administrative interfaces exist as disconnected islands. Without integration, feedback cannot flow to where decisions are made. Data in one system remains invisible to another.

Systemic Root Causes
  • No API bridges between feedback collection and TMS
  • Mismatched data schemas and identifier systems
  • No synchronization strategy or ETL processes
Principals, district officers lack unified visibility across data sources
04

The Volume Overwhelm

Manual processing cannot scale with feedback volume.

Thousands of feedback entries arrive each collection cycle. Administrative staff have finite hours. Without automated processing, the backlog grows continuously. By the time any fraction is reviewed, the information is stale.

Systemic Root Causes
  • No batch automation for high-volume entry processing
  • Human reviewers limited to dozens of entries per day
  • Peak load periods (semester-end) create insurmountable backlogs
Administrators spend excessive time on manual review with diminishing returns
05

The Timeliness Trap

By the time insights arrive, intervention windows have closed.

Batch processing and periodic reviews create delays of days to weeks between when students share feedback and when institutions learn of issues. Mid-semester feedback may not surface until the term ends. Every insight arrives after intervention is possible.

Systemic Root Causes
  • No real-time processing infrastructure exists
  • Batch review cycles measured in weeks, not hours
  • No alerting or notification system for urgent issues
Proactive intervention becomes impossible; institutions react only to past failures
06

The Connectivity Barrier

Rural networks fragment data before it can be processed.

ITIs in rural Krishna District face intermittent internet connectivity. Without offline-capable systems and robust synchronization, feedback submissions fail silently, data arrives incomplete, and time-sensitive insights become stale during transmission delays.

Systemic Root Causes
  • No offline-first architecture for feedback collection
  • No queuing mechanism for delayed sync when connectivity returns
  • Device constraints for students with limited smartphone access
Feedback from remote ITIs arrives incomplete or not at all
07

The Privacy/Consent Friction

Compliance requirements without compliance infrastructure.

DPDP Act compliance requires consent management, data minimization, anonymization, and audit trails. Without these mechanisms, institutions face a choice: share data freely and risk violations, or restrict sharing completely and lose analytical value. Neither path serves students.

Systemic Root Causes
  • No consent capture and tracking mechanism exists
  • No anonymization pipeline for aggregated analytics
  • No access control matrix for role-based data visibility
Data sharing blocked by compliance uncertainty; faculty performance data inaccessible
08

The Governance Vacuum

Even if insights existed, no one owns the action.

The feedback-to-action loop has no defined owner. Who decides when to intervene? Who is accountable for acting on insights? Without decision rights, escalation paths, or action SLAs, insightseven if perfectly generatedwould have nowhere to go and no one to act on them.

Systemic Root Causes
  • No defined decision thresholds for intervention triggers
  • No escalation matrix from insight to action owner
  • No feedback-to-closure workflow or accountability
Insights generated but never acted upon; improvement loop remains open indefinitely

Detailed Process Steps

01

Access the Feedback App

Student opens the Progressive Web App (PWA) via multiple entry points: scanning a QR code posted in classrooms, clicking a WhatsApp link shared by faculty, or tapping the app icon on their home screen. The app loads instantly from cache even without internet connectivity, presenting a fully localized Telugu interface optimized for low-end Android devices.

PWA (React) Service Worker IndexedDB Cache
02

Authenticate via Mobile OTP

Student enters their registered mobile number and receives a one-time password via SMS. Upon successful authentication, a session token is established and the app loads their enrolled trades, current faculty assignments, and class schedule from the cached roster data synced from ITI TMS (Training Management System).

Keycloak IAM SMS Gateway JWT Tokens ITI TMS API
03

Record Voice Feedback in Telugu

Student taps the microphone button and speaks their feedback naturally in Telugu, Tenglish (code-mixed Telugu-English), or pure English. The voice is recorded locally with Opus audio compression for efficient storage. A "Success" confirmation appears immediately while the audio is queued for transcription. Bhashini STT (Speech-to-Text) converts the voice to text with >90% accuracy for regional dialects.

Bhashini STT API Web Audio API Opus Codec Background Sync
04

Complete Structured Rating Questions

After voice input, the student answers optional structured questions using intuitive emoji-based scales (1-5 ratings) for sentiment and comprehension level. These structured ratings are captured alongside the open-text feedback to enable quantitative trend analysis. The UI adapts to student literacy levels with visual prompts and Telugu labels.

React Components Form Validation Accessibility (WCAG)
05

Submit & Receive Closure Notification

Feedback is submitted and queued in IndexedDB for sync when connectivity is available. The submission is anonymized during ingestion (K-anonymity applied). When an action is taken based on their feedback, the student receives a "You asked, we fixed" notification via WhatsApp or in-app pushbuilding trust and encouraging continued participation in the feedback loop.

IndexedDB Queue K-Anonymity Engine WhatsApp Business API Firebase Push

Detailed Process Steps

01

Receive Weekly Performance Digest

Every Friday at 4PM, faculty receives a WhatsApp message containing a personalized weekly digest with a secure magic link. This timing is optimized for end-of-week reflection. No app installation or password memorization requiredthe link contains a time-limited JWT token for authentication.

WhatsApp Business API Gupshup Gateway JWT Magic Link
02

Access Personal Dashboard

Tapping the magic link opens a mobile-optimized web dashboard authenticated via the embedded token. Role-Based Access Control (RBAC) ensures faculty sees ONLY their own performance datano peer comparisons or rankings that could create unhealthy competition. The dashboard loads instantly with pre-cached insights.

Faculty Dashboard (React) Keycloak RBAC Row-Level Security
03

Review Sentiment Trends & Themes

Dashboard displays aggregated sentiment trends over time (positive/negative/neutral) with drill-down into specific themes. Example: "60% of students report difficulty understanding safety procedures in welding class." Anonymized student quotes provide context without revealing identities. Trend arrows show week-over-week changes.

PostgreSQL Analytics Apache ECharts K-Anonymity Display
04

Read AI Coaching Recommendations

LLM-generated coaching suggestions provide actionable improvement ideas grounded in actual feedback. Example: "Consider more hands-on demonstration of PPE usage before practical sessions." Each recommendation cites source feedback (anonymized) and links to relevant teaching resources. Suggestions are developmental, not punitive.

LLM RAG Pipeline IndicBERT + GPT-4 Source Citations
05

Flag Inaccurate Insights for Review

If faculty disagrees with an AI-generated insight, they can flag it with a single tap and optionally provide context. Flagged items are routed to a human review queue where administrators validate the concern. This feedback loop triggers model fine-tuning to improve future accuracycreating a continuously improving system.

Human Review Queue Model Feedback Loop Audit Logging
